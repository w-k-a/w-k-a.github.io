JSON什么，优缺点

什么是json,优缺点

有一种叫做JSON(javascript object nonation)的轻量级数据交换格式能够代替xml的工作，就是json。

数据格式比较简单，易于读写，格式都是压缩的，占用宽带小。

易于解析这种语言，客服端可以通过Javascript可以简单的通过eval()进行json数据的读取。包括ActionScript,C,C#,ColdFusion,Java,JavaScrpt,perl,PHP,Python,ruby等语言服务器端语言，便于服务器端的解析。各语言对JSON支持特别好，自从Ajax的流行，json格式的传输就更流行了。

面试时：可以这样说：json是一个字符串，有规则的字符串，然后表达形式是以键值对的形式存在的，可以用来前后端的交互，接口口，配置文件，文件存储，目前移动端比较火，一般的项目也会用到json来传输。

优点：

数据格式比较简单，易于读写，格式都是压缩的，占用宽带小；

易于解析，客服端JavaScript可以通过简单的eval()进行json数据的读取；

支持多种语言，包括ActionScript, C, C#, ColdFusion, Java, JavaScript, Perl, PHP, Python, Ruby等服务器端语言，便于服务器端的解析；

在PHP世界，已经有PHP-JSON和JSON-PHP出现了，偏于PHP序列化后的程序直接调用，PHP服务器的对象、数组等能直接生成JSON格式，便于客服端的访问提取；

因为JSON格式能直接为服务器代码使用，大大简化了服务器端和客服端的代码开发量，且完成任务不变，并且易于维护。

缺点

  1. 没有xml格式这么推广的深入人心和喜用广泛，没有xml那么通用性；
  2. json格式目前在web Service中的推广还属于初级阶段。
